<动态场景人口系统>

## 激活条件
- 地点变化：强制激活，生成初始NPC
- 周期刷新：每15分钟或关键时间点（上课铃、午休结束）
- 事件驱动：新NPC进入空场景时立即激活
- 天气变化：若天气导致人流密度数值变化，刷新场景总人数与NPC出现概率

## 人流密度数值接口
人流密度使用0~5数值。先根据场景/地点取得人口密度基准，再叠加 `天气场景修正[mvu_plot]` 输出的“人口密度修正”，得到最终人流密度。

| 人流密度 | 含义 | NPC生成倾向 |
|---|---|---|
| 0 | 无人 | 不生成随机路人NPC；只保留固定NPC/剧情指定NPC |
| 1 | 极低 | 只生成少量偶遇NPC；第三方出现概率很低 |
| 2 | 低 | 少量路人；可形成半公开风险 |
| 3 | 中 | 正常公共场景人流 |
| 4 | 高 | 人流密集；公开风险显著上升 |
| 5 | 极高 | 拥挤场景；持续旁观、误入镜头、干预风险最高 |

## NPC数量计算

基础公式：场景总人数 = $(roll(xdy+z)) + 时间修正值 + 最终人流密度对应数量修正

人流密度修正值：
| 人流密度 | 数量修正 | 说明 |
|---|---:|---|
| 0 | 强制为0 | 无随机路人NPC |
| 1 | -2d3 | 极低人流，最低可降至0 |
| 2 | -1d3 | 低人流，减少路人数量 |
| 3 | +0 | 正常人流 |
| 4 | +1d6 | 高人流，增加路人数量 |
| 5 | +2d6 | 极高人流，显著增加路人数量 |

最低值规则：
- 人流密度=0：随机NPC总数=0。
- 人流密度=1：随机NPC总数最低=0。
- 人流密度≥2：随机NPC总数最低=1，除非地点本身明确无人/封闭/废弃。

示例：
| 场景 | 基础投骰 | 时间修正 | 人流密度计算 |
|------|----------|----------|----------|
| CBD服装店 | 2d6+3 | - | 场景基准 + 天气人口密度修正 |
| 废弃工厂 | 1d6+1 | - | 场景基准通常0~1，再叠加天气修正 |
| 考试周图书馆 | 基础值 | +3d6 | 室内固定人流可覆盖天气负修正 |

最低保障：有明确类型的NPC（观察欣赏型、社交接触型、直接试探型）至少占场景总人数的30%。若随机NPC总数≤2，可只生成1名明确类型NPC。

## 天气联动规则
天气不再直接给出最终人流密度，只提供人口密度修正值：
- 晴朗：人口密度修正+0。
- 多云：人口密度修正+0。
- 阴天：人口密度修正-1。
- 小雨：人口密度修正-1。
- 中雨：人口密度修正-2。
- 雷阵雨：人口密度修正-3；避雨点/门厅/车站可由场景活动修正抵消或反向提高。
- 雾：人口密度修正-2；近距离突然出现风险提高。

## 场景基准人流覆盖
生成NPC数量前，先参考 `场景判定[mvu_plot]` 的“场景基准四轴表”确定人口密度基准，再叠加天气±修正、时间和事件修正。

场所基准优先级：
1. 地点条目中明确写出当前人数/固定NPC/活动状态 → 优先采用地点条目。
2. 当前叙事明确出现排队、上课、集会、避雨、散场等状态 → 优先采用叙事状态。
3. 若无明确人数，则按 `场景判定[mvu_plot]` 的场景基准四轴表取人口密度基准。
4. 最后叠加 `天气场景修正[mvu_plot]` 的人口密度修正和时间段修正。

常用场所人口密度基准：
| 场所类型 | 人口密度基准 | NPC生成说明 |
|---|---:|---|
| 教室/课堂/考试场 | 4~5 | 固定高人流，天气通常不清空 |
| 食堂/商业街/商场主通道 | 4~5 | 高人流，公开风险强 |
| 车站/候车区/门厅/避雨点 | 3~5 | 恶劣天气可因避雨聚集而升高 |
| 便利店角落/货架后/商场边角 | 2~3 | 人流不低，但可见场景人数可能为0 |
| 走廊拐角/楼梯间/偏僻楼层 | 2~3 | 遭遇风险中等 |
| 公园角落/小树林边缘/操场偏角 | 1~3 | 依时间和路径调整 |
| 小树林深处/小巷死角/深夜公园角落 | 1~2 | 低人流，少量偶遇 |
| 废弃建筑/旧仓库/地下管廊 | 0~1 | 通常不生成或仅生成极少偶遇NPC |
| 锁门房间/私人住处/独处空间 | 0 | 不生成随机路人NPC |
| NPC同伙知情配合在场 | 0~2 | 同伙按固定NPC处理，不计入随机路人 |

覆盖规则：
- 课堂/考试/集会/排队窗口/演唱会：人口密度可固定为4~5。
- 商场室内/车站候车区/避雨门厅：恶劣天气下人口密度可升至3~5。
- 废弃建筑/锁门房间/私人住处：人口密度可固定为0~1。
- 雾天或暴雨只影响户外开放区域，不自动清空室内固定人流。
- 人口密度只决定潜在NPC数量与遭遇概率；是否形成旁观者风险，仍需由 `场景判定[mvu_plot]` 单独计算可见场景人数。

## NPC倾向类型

| 类型 | 行为特征 | 通用占比 |
|------|----------|----------|
| 观察欣赏型 | 多看、回头看、保持距离 | 15% |
| 社交接触型 | 搭讪、要联系方式 | 40% |
| 直接试探型 | 坦诚表达兴趣、直接试探、渐进推进 | 35% |
| 无视忽略型 | 完全不关注 | 10% |

## 人流密度对NPC倾向的修正
| 人流密度 | 倾向修正 |
|---|---|
| 0 | 不生成随机倾向 |
| 1 | 观察欣赏型+10%，社交接触型-10%，直接试探型-10%；少量NPC更谨慎 |
| 2 | 观察欣赏型+5%，直接试探型-5%；互动更依赖遮挡和时机 |
| 3 | 使用通用占比 |
| 4 | 社交接触型+5%，直接试探型+5%，无视忽略型-10%；误触和搭话机会增加 |
| 5 | 社交接触型+10%，直接试探型+10%，无视忽略型-15%；拥挤导致接触机会和旁观风险同时上升 |

## 场景概率表 (d100)

通用场景（安全公共场所）：
1-15: 观察欣赏型 | 16-55: 社交接触型 | 56-90: 直接试探型 | 91-100: 无视忽略型

## 输出格式

NPC列表包含：生成时间、总数、人流密度数值、每个NPC的姓名/倾向/特征

人数多时使用简称：
> 人流密度4；12位普通社员A-J（观察欣赏型2名，社交接触型5名，直接试探型5名）

人数极低时使用明确提示：
> 人流密度1；场景内仅1名路人NPC，行动匆忙，短暂停留。

无人时使用明确提示：
> 人流密度0；无随机路人NPC，仅保留固定NPC/剧情指定NPC。

## 特殊规则

固定NPC：
- 不参与随机生成，预设倾向和人设
- 可重复出现，列表中标记为"固定NPC"
- 不受人流密度=0的随机清空影响

无延伸机制：
- NPC不留联系方式
- 同一NPC不会在不同场景再次出现
