<服饰描述编写指南>

## 存储格式

服饰描述存储在变量的服装状态键值中，格式为：
```yaml
分类:
  服饰名称: >-
    参数化描述字符串
```

示例：
```yaml
上装:
  白色短袖圆领T恤: >-
    涤棉混纺(65/35),厚0.8mm,低弹.设计基准:胸围差13/腰围60.圆领,口深8cm宽18cm.袖至上臂中段,口宽16cm.衣长58cm,摆宽52cm.(暴露值:5)
```

### 穿着状态描述

在穿着状态下，可在参数化描述字符串末尾追加`[状态:...]`标记，用于描述服饰当前的实时动态变化。状态描述不改变服饰的固有参数，仅叠加当前穿着情境下的临时效果。

格式：
```yaml
分类:
  服饰名称: >-
    参数化描述字符串[状态:状态1;状态2;...]
```

示例：
```yaml
上装:
  白色短袖圆领T恤: >-
    涤棉混纺(65/35),厚0.8mm,低弹.设计基准:胸围差13/腰围60.圆领,口深8cm宽18cm.袖至上臂中段,口宽16cm.衣长58cm,摆宽52cm.(暴露值:5)[状态:被撩至胸线以上;前摆被扎入裙腰]
下装:
  黑色包臀裙: >-
    棉氨混纺(95/5),厚0.6mm,低弹.设计基准:臀围86/腰围60.自然腰线.裙长48cm,摆宽42cm.(暴露值:8)[状态:后摆被风吹起至臀部以上]
内衣:
  白色全罩杯文胸: >-
    棉质,厚1.2mm,低弹.设计基准:胸围差13.全罩杯.肩带宽1.5cm,下围带宽3cm.无装饰.(暴露值:0)[状态:左肩带滑落至上臂;罩杯被拉低至乳房下沿]
```

#### 状态描述编写规则

1. **仅描述变化事实**：用客观、精确的语句描述服饰当前发生了什么变化，不描述效果或感受
   - ✅ `被撩至胸线以上`
   - ✅ `右侧肩带断裂`
   - ✅ `前片被拉至一侧`
   - ❌ ~~`性感地被撩起`~~（禁止主观形容）
   - ❌ ~~`露出了内衣`~~（禁止描述效果，效果由LLM根据参数推算）

2. **使用位置+动作格式**：[部位][变化动词][参照位置/程度]
   - `后摆被掀至腰线` — 部位:后摆, 动作:被掀, 参照:腰线
   - `左罩杯被拉低2cm` — 部位:左罩杯, 动作:被拉低, 程度:2cm
   - `裆部面料被拉向左侧3cm` — 部位:裆部面料, 动作:被拉向左侧, 程度:3cm

3. **多状态用分号分隔**：按发生顺序排列，每个状态独立描述
   - `[状态:左肩带滑落至上臂;领口被拉低至乳沟下方]`

4. **状态可叠加、可移除**：
   - 添加状态：在`[状态:...]`中追加新的分号分隔项
   - 移除状态：删除对应项（如衣服被整理回原位）
   - 清空所有状态：移除整个`[状态:...]`标记，恢复为纯参数描述
   
5. **状态不改变固有参数**：状态描述是临时叠加层，不修改面料、尺寸、暴露值等固有参数。LLM应将状态描述与固有参数结合推算当前实际穿着效果

#### 常见状态描述词汇参考

位移类：
- `被撩至[位置]` / `被掀至[位置]` / `被拉至[位置]`
- `上移Xcm` / `下滑至[位置]` / `滑落至[位置]`
- `被扎入[位置]` / `被塞入[位置]`

损坏类：
- `[部位]撕裂Xcm` / `[部位]断裂` / `[部位]脱线`
- `[部位]纽扣脱落` / `拉链滑开至[位置]`

形变类：
- `[部位]被拉伸变形` / `[部位]被扯松`
- `整体褶皱严重` / `[部位]被折叠`

附着类：
- `[部位]沾染[物质]` / `[部位]浸湿`
- `被[物质]打湿[范围]`

穿法类：
- `反穿` / `只穿一侧` / `挂在[位置]`
- `前后穿反` / `[部位]未扣合`

## 参数编写格式

将所有参数压缩为单行字符串，用逗号分隔，按以下顺序排列：

### 上装
```
[面料成分],厚[X]mm,[弹性].设计基准:胸围差[X]/腰围[X].[领口类型],口深[X]cm宽[X]cm.袖[描述],口宽[X]cm.衣长[X]cm,摆宽[X]cm.[外观特征(如有)].[隐藏特性(如有)].(暴露值:[X])
```

### 下装
```
[面料成分],厚[X]mm,[弹性].设计基准:臀围[X]/腰围[X].[腰线位置].裙长/裤长[X]cm,摆宽/脚宽[X]cm.[外观特征(如有)].[隐藏特性(如有)].(暴露值:[X])
```

### 内衣
```
[面料成分],[弹性].设计基准:胸围差[X].[罩杯覆盖类型].肩带宽[X]cm,下围带宽[X]cm.[装饰].[外观特征(如有)].[隐藏特性(如有)].(暴露值:[X])
```

### 内裤
```
[面料成分],[弹性].设计基准:臀围[X].[款式].前片宽[X]cm,后片[覆盖类型].[腰线高度].[外观特征(如有)].[隐藏特性(如有)].(暴露值:[X])
```

### 袜子
```
[面料成分].长[X]cm,[厚度/旦数].袜口[松紧].[外观特征(如有)].(暴露值:[X])
```

### 鞋子
```
[款式].跟高[X]cm,[鞋头形状],[固定方式].[外观特征(如有)].(暴露值:[X])
```

## 参数说明

### 弹性
无弹/低弹/中弹/高弹

### 设计基准
衣服设计适配的身材数值（女性M码标准）。LLM通过角色实际数据与设计基准的差值，依照<身材与服饰关联描述>推算穿着效果。

女性M码标准参数（未特别说明时所有服饰默认使用此基准）:
- 胸围差基准: 13cm (B罩杯, 胸围差范围: A=9~11, B=12~14, C=15~16, D=17~19, E=20~21, F=22~23)
- 臀围基准: 86cm (翘臀起点, 范围: 骨感80~85, 翘臀85~92, 丰满93~98)
- 腰围基准: 60cm (A4腰, 范围: 骨感50~55, A4腰56~63, 少女腰64~70, 微胖71~79)

### 透明度
不在服饰描述中标注。LLM根据面料成分和厚度，依照<身材与服饰关联描述>中的面料透明度推断规则自行判断。

### 外观特征
用于记录**不受身材数据影响、无法从面料/厚度/弹性等基础参数推算的可见设计特色**。
用逗号分隔多项特征，直接写在隐藏特性之前的位置。

适用: 纯视觉/装饰性设计元素
- `外观:下摆渐变色(白→浅粉)` — 颜色渐变
- `外观:胸前蕾丝拼贴,袖口蕾丝花边` — 蕾丝装饰
- `外观:侧缝镂空编织带` — 镂空/剪裁设计
- `外观:前胸刺绣玫瑰图案` — 印花/刺绣
- `外观:腰部系带蝴蝶结` — 系带/绑带装饰
- `外观:后背交叉绑带露背` — 特殊剪裁结构
- `外观:袜口蕾丝花边3cm` — 袜子装饰

不适用（已在其他参数中体现，禁止重复写）:
- ~~外观:面料很薄~~ → 由厚度参数体现
- ~~外观:穿上很紧~~ → 由身材差值推算
- ~~外观:领口很低~~ → 由领口深度参数体现

### 隐藏特性
仅用于**无法从面料、厚度、弹性、身材差值等参数推算出的人为特殊属性**。
用"隐藏:"前缀标记。穿着者不知情，剧情中可安排触发。

适用: 人为加工/改造/机关类属性
- `隐藏:裆部面料经特殊打磨,摩擦力是其余部位3倍`
- `隐藏:内置微型震动器,可通过手机遥控启动`
- `隐藏:肩带连接处经弱化处理,猛拉即断`

不适用（可从参数推算，禁止写为隐藏）:
- ~~隐藏:蹲下时臀缝处撑薄至半透~~ → 由面料厚度+臀围差值+姿态规则推算
- ~~隐藏:强光下近乎透明~~ → 由面料成分+厚度+颜色推算
- ~~隐藏:出汗后变透明~~ → 由面料成分推算

## 编写原则

### 固有参数编写原则
- 所有数值用阿拉伯数字+cm，不用文字描述长短
- 设计基准必填，这是差值推算的基础
- 面料成分精确到百分比，厚度必填（透明度由此推断）
- 一件衣服 = 一个键值对，描述压缩为单字符串
- 外观特征仅记录可见设计特色，不描述穿着效果
- 禁止在描述中预写特定身材的穿着效果
- 禁止使用模糊形容词（"很紧""比较宽松"）
- 禁止显式标注透明度（由面料+厚度推断）

### 状态描述编写原则
- 状态描述是可选的，仅在服饰发生动态变化时添加
- 状态描述追加在参数字符串末尾的`[状态:...]`中，不修改前面的固有参数
- 仅描述客观变化事实，禁止描述效果/感受/主观评价
- 使用精确的位置和程度描述，禁止模糊表达
- 当服饰恢复原始穿着状态时，移除`[状态:...]`标记
- LLM应结合固有参数+状态描述+身材数据综合推算当前实际穿着效果
